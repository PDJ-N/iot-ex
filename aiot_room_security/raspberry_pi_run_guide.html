Raspberry Pi 실행 설명서

이 문서는 aiot_room_security 프로젝트를 라즈베리파이에 옮기고, 하드웨어 회로를 연결한 뒤 실제 침입 감지 프로그램을 실행하는 절차를 정리한다.

프로젝트: AI 기반 개인 방 침입 감지 실행 파일: main.py 실제 모드: AIOT_MOCK_MODE=0 PIR GPIO16 / LED GPIO21 / 부저 GPIO18

1. 전체 실행 흐름

1단계 회로 연결: PIR, LED, 저항, 부저, 웹캠 연결
2단계 코드 준비: GitHub clone 또는 폴더 복사
3단계 모델과 텔레그램 설정: MobileNet SSD, bot token, chat_id
4단계 실제 실행: AIOT_MOCK_MODE=0 후 python main.py
핵심: 이 프로젝트는 하드웨어 없이 PC에서 mock 테스트할 수 있지만, 실제 PIR 센서, LED, 부저, 웹캠을 작동하려면 반드시 Raspberry Pi에서 실행해야 한다.

2. 라즈베리파이 준비물

  • Raspberry Pi 4 또는 Raspberry Pi 5
  • Raspberry Pi OS 64-bit 권장
  • 인터넷 연결
  • USB 웹캠
  • 브레드보드, PIR 센서, 빨간 LED, 330Ω 저항, 능동부저
  • 텔레그램 계정과 BotFather로 만든 봇
회로 연결은 라즈베리파이 전원을 끈 상태에서 먼저 끝낸다. 전원이 켜진 상태에서 GPIO 배선을 바꾸면 보드가 손상될 수 있다.

3. 회로 연결 확인

장치 연결
PIR Signal GPIO 16, 물리 36번 핀
PIR VCC 3.3V, 물리 1번 핀
PIR GND GND, 물리 6번 또는 39번 핀
빨간 LED 긴 다리 GPIO 21, 물리 40번 핀
빨간 LED 짧은 다리 330Ω 저항을 거쳐 GND
3핀 능동부저 S GPIO 18, 물리 12번 핀
3핀 능동부저 + 3.3V, 물리 1번 핀
3핀 능동부저 - GND

그림 설명서는 circuit_guide.html을 열어 확인한다.

4. 프로젝트를 라즈베리파이로 옮기는 방법

방법 A: GitHub에 올린 뒤 clone

cd ~
git clone <본인의_GitHub_저장소_URL>
cd <저장소_폴더>/aiot_room_security

방법 B: PC에서 scp로 폴더 복사

Windows PowerShell에서 실행한다. 라즈베리파이 IP는 본인 환경에 맞게 바꾼다.

scp -r .\aiot_room_security pi@192.168.0.10:~/
제출용 GitHub 링크가 필요하므로, 최종적으로는 GitHub 저장소에 올리고 라즈베리파이에서 git clone하는 방식을 권장한다.

5. Python 환경 설치

라즈베리파이 터미널에서 실행한다.

cd ~/aiot_room_security
sudo apt update
sudo apt install -y python3-venv python3-pip git curl
python3 -m venv .venv --system-site-packages
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
--system-site-packages를 사용하는 이유는 라즈베리파이에 기본 설치된 GPIO 관련 패키지와 충돌을 줄이기 위해서이다.

6. OpenCV DNN 모델 파일 준비

수업 Section 16에서 사용한 MobileNet SSD 모델 파일 2개가 필요하다.

cd ~/aiot_room_security
git clone https://github.com/moonchuljang/OpencvDnn.git /tmp/OpencvDnn
mkdir -p models
cp /tmp/OpencvDnn/models/frozen_inference_graph.pb models/
cp /tmp/OpencvDnn/models/ssd_mobilenet_v2_coco_2018_03_29.pbtxt models/
ls -l models

정상이라면 아래 두 파일이 보인다.

frozen_inference_graph.pb
ssd_mobilenet_v2_coco_2018_03_29.pbtxt

7. 텔레그램 봇 설정

  1. 텔레그램에서 BotFather를 검색한다.
  2. /newbot 명령으로 봇을 만들고 token을 복사한다.
  3. 새 봇 채팅방에 들어가 아무 메시지나 보낸다.
  4. 라즈베리파이 터미널에서 아래 명령으로 chat_id를 확인한다.
export TELEGRAM_BOT_TOKEN="BotFather에서_받은_token"
curl "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/getUpdates"

출력 JSON에서 "chat":{"id":...} 값을 찾아 설정한다.

export TELEGRAM_CHAT_ID="확인한_chat_id"
token은 비밀번호처럼 취급한다. 코드나 GitHub에 직접 저장하지 말고 환경변수로 넣는다.

8. 하드웨어 단품 테스트

전체 프로그램을 실행하기 전에 LED, 부저, PIR, 웹캠을 따로 확인한다.

LED와 부저 테스트

cd ~/aiot_room_security
source .venv/bin/activate
export AIOT_MOCK_MODE=0
python - <<'PY'
import time
import hardware

hardware.led_on()
hardware.buzzer_on()
time.sleep(1)
hardware.all_off()
print("LED와 부저 테스트 완료")
PY

PIR 센서 테스트

cd ~/aiot_room_security
source .venv/bin/activate
export AIOT_MOCK_MODE=0
python - <<'PY'
import time
import hardware

for i in range(20):
    print("PIR value:", int(hardware.read_pir_motion()))
    time.sleep(0.5)
hardware.all_off()
PY

웹캠 확인

ls /dev/video*
python - <<'PY'
import cv2

camera = cv2.VideoCapture(0)
print("camera opened:", camera.isOpened())
ok, frame = camera.read()
print("frame read:", ok, "shape:", None if frame is None else frame.shape)
camera.release()
PY
웹캠이 0번으로 열리면 실제 실행 전에 export AIOT_CAMERA_INDEX=0을 추가한다.

9. 실제 프로그램 실행

회로, 모델, 텔레그램 설정이 끝난 뒤 실행한다.

cd ~/aiot_room_security
source .venv/bin/activate
export AIOT_MOCK_MODE=0
export AIOT_CAMERA_INDEX=0
export TELEGRAM_BOT_TOKEN="BotFather에서_받은_token"
export TELEGRAM_CHAT_ID="확인한_chat_id"
python main.py

종료할 때는 터미널에서 Ctrl+C를 누른다. 프로그램의 finally 코드가 LED와 부저를 꺼준다.

정상 동작 기준: 움직임 감지 → AI person 탐지 → CSV 로그 저장 → 텔레그램 메시지 전송 → LED 점멸과 부저 경고.

11. 자주 나는 오류

오류 처리 방법
gpiozero 오류 가상환경 활성화 후 pip install gpiozero를 다시 실행한다.
모델 파일 없음 models/ 폴더에 .pb, .pbtxt 두 파일이 있는지 확인한다.
웹캠 열기 실패 AIOT_CAMERA_INDEX=0 또는 다른 번호를 시험한다.
텔레그램 전송 실패 token, chat_id, 인터넷 연결, 봇에게 먼저 메시지를 보냈는지 확인한다.
LED와 부저가 계속 켜짐 Ctrl+C 종료 후에도 켜져 있으면 배선을 빼고 hardware.all_off() 테스트를 실행한다.

12. Codex 웹으로 라즈베리파이에서 구동 가능한가?

결론: Codex 웹 또는 Codex cloud만으로는 라즈베리파이의 실제 GPIO 핀, PIR 센서, USB 웹캠, 부저를 직접 제어하는 실행 검증을 할 수 없다. 실제 하드웨어 실행은 라즈베리파이 터미널에서 해야 한다.

공식 Codex 매뉴얼 기준으로 Codex web은 GitHub 계정을 연결해 저장소 코드를 작업하고 PR을 만들 수 있는 환경이다. Codex cloud task는 저장소를 컨테이너에 체크아웃하고 setup script를 실행한 뒤 코드를 수정·검증하는 방식이다. 이 컨테이너는 실제 라즈베리파이에 연결된 GPIO나 USB 웹캠을 갖고 있지 않다.

다만 Codex App의 원격 연결 기능을 SSH host에 연결하면, 연결된 호스트의 파일과 shell에서 작업할 수 있다. 라즈베리파이를 SSH host로 구성하고 Codex가 그 환경에 접근하도록 별도로 설정하면 원격 shell 작업은 가능할 수 있다. 그래도 보안상 실제 GPIO 실행은 사람이 회로 상태를 보고 승인하면서 진행하는 것이 안전하다.

방식 라즈베리파이 실제 하드웨어 실행 가능 여부
Codex web / cloud task 불가능. 코드는 만들 수 있지만 실제 GPIO와 웹캠이 있는 환경이 아니다.
현재 PC의 Codex 작업공간 불가능. Windows 작업공간에서는 mock 테스트만 가능하다.
라즈베리파이 터미널 직접 실행 가능. 가장 권장하는 방식이다.
Codex App + SSH host로 라즈베리파이 연결 환경을 제대로 구성하면 shell 작업은 가능할 수 있다. 단, 회로 안전 확인은 사람이 해야 한다.

참고한 공식 문서 항목: Codex manual의 Codex web, Cloud environments, Remote connections.
