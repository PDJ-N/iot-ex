회로 연결 후 실행 순서

PIR 센서, 3핀 부저, USB 웹캠 연결이 끝난 뒤 라즈베리파이에서 무엇을 해야 하는지 순서대로 정리한 설명서이다. 먼저 부품별 단품 테스트를 하고, 그 다음 전체 프로그램을 실행한다.

1. 코드 받기 2. 환경 설치 3. 부저/PIR/웹캠 테스트 4. AI 모델 준비 5. main.py 실행

1. 전체 순서

1단계 GitHub에서 프로젝트 받기
2단계 Python 가상환경 설치
3단계 부저, PIR, 웹캠 단품 테스트
4단계 MobileNet SSD 모델 파일 준비
5단계 실제 프로그램 실행
회로 연결 후 바로 main.py를 실행하지 말고, 부저와 PIR 센서가 따로 잘 동작하는지 먼저 확인한다.

2. GitHub에서 코드 받기

라즈베리파이 터미널을 열고 아래 명령어를 입력한다.

cd ~
git clone https://github.com/PDJ-N/iot-ex.git
cd iot-ex/aiot_ai_room_security

이미 한 번 clone 했다면 새로 clone하지 말고 업데이트만 한다.

cd ~/iot-ex
git pull
cd aiot_ai_room_security

3. Python 환경 설치

프로젝트 폴더 안에서 실행한다.

python3 -m venv .venv --system-site-packages
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
다음부터는 이 폴더에 들어온 뒤 항상 source .venv/bin/activate를 먼저 실행한다.

4. 부저 테스트

부저가 1초 동안 울리면 정상이다.

cd ~/iot-ex/aiot_ai_room_security
source .venv/bin/activate
export AIOT_MOCK_MODE=0

python - <<'PY'
import time
import buzzer

buzzer.buzzer_on()
time.sleep(1)
buzzer.buzzer_off()
print("부저 테스트 완료")
PY
3핀 부저에서 S는 GPIO18, +는 3.3V, -는 GND이다. +를 GPIO18에 꽂지 않는다.

5. PIR 센서 테스트

센서 앞에서 움직이면 PIR: 1, 움직임이 없으면 PIR: 0이 나와야 한다.

cd ~/iot-ex/aiot_ai_room_security
source .venv/bin/activate
export AIOT_MOCK_MODE=0

python - <<'PY'
import time
import sensor

for i in range(20):
    print("PIR:", int(sensor.read_motion()))
    time.sleep(0.5)
PY
PIR 센서는 전원을 켠 직후 안정화 시간이 필요할 수 있다. 값이 이상하면 30초 정도 기다렸다가 다시 확인한다.

6. USB 웹캠 테스트

먼저 라즈베리파이가 웹캠 장치를 인식했는지 확인한다.

ls /dev/video*

그 다음 OpenCV로 프레임을 읽어 본다.

cd ~/iot-ex/aiot_ai_room_security
source .venv/bin/activate

python - <<'PY'
import cv2

cam = cv2.VideoCapture(0)
print("camera opened:", cam.isOpened())
ok, frame = cam.read()
print("frame read:", ok)
print("shape:", None if frame is None else frame.shape)
cam.release()
PY
camera opened: True와 frame read: True가 나오면 웹캠은 정상이다.

7. AI 모델 파일 준비

수업에서 사용한 OpenCV DNN MobileNet SSD 모델 파일 두 개를 준비한다. GitHub 저장소를 검색하거나 clone하지 않고, 필요한 파일만 직접 다운로드한다. 

cd ~/iot-ex/aiot_ai_room_security
python3 download_models.py
ls -lh models

아래 두 파일이 보이면 정상이다.

frozen_inference_graph.pb
ssd_mobilenet_v2_coco_2018_03_29.pbtxt

이미 수업 때 받은 모델 파일이 있다면 다운로드 대신 아래처럼 직접 복사해도 된다.

mkdir -p models
cp /모델파일이있는경로/frozen_inference_graph.pb models/
cp /모델파일이있는경로/ssd_mobilenet_v2_coco_2018_03_29.pbtxt models/
ls -lh models

8. 텔레그램 없이 먼저 전체 실행

텔레그램 설정 전에 먼저 전체 프로그램이 부저, 카메라, AI, 로그까지 도는지 확인한다. token과 chat_id가 없으면 텔레그램만 건너뛴다. 

cd ~/iot-ex/aiot_ai_room_security
source .venv/bin/activate

export AIOT_MOCK_MODE=0
export AIOT_CAMERA_INDEX=0

python main.py

사람이 PIR 앞에서 움직이면 아래 흐름이 나와야 한다.

  • 콘솔에 PIR 감지 메시지 출력
  • AI person confidence 출력
  • 부저 울림
  • captures/ 폴더에 사진 저장
  • logs/event_log.csv에 로그 기록

9. 텔레그램 설정 후 최종 실행

BotFather에서 받은 token과 chat_id를 넣고 실행한다.

cd ~/iot-ex/aiot_ai_room_security
source .venv/bin/activate

export AIOT_MOCK_MODE=0
export AIOT_CAMERA_INDEX=0
export TELEGRAM_BOT_TOKEN="BotFather에서_받은_token"
export TELEGRAM_CHAT_ID="확인한_chat_id"

python main.py
정상 동작하면 텔레그램으로 “방에 사람이 감지되었습니다.” 메시지와 사진이 전송된다.

10. 결과 확인 명령어

확인할 것 명령어
로그 파일 cat logs/event_log.csv
저장 사진 ls -l captures
모델 파일 python3 download_models.py 실행 후 ls -lh models
웹캠 장치 ls /dev/video*
