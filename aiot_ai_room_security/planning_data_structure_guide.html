AIoT 방 침입 감지 시스템 기획 및 데이터 구조도

Raspberry Pi 5, PIR 센서, USB 웹캠, OpenCV DNN MobileNet SSD, 능동 부저, 텔레그램 봇 API, CSV 로그를 결합한 기말 프로젝트의 설계 설명서이다. 보고서에 바로 옮겨 쓸 수 있도록 시스템 목적, 데이터 구조, 처리 흐름, 모듈 역할을 정리했다.

프로젝트 기획 데이터 구조도 모듈 구조 CSV 로그 구조 보고서 참고용

1. 프로젝트 기획 개요

이 시스템은 방 안에 누군가 들어오는 상황을 감지하기 위한 Raspberry Pi 기반 AIoT 보안 프로젝트이다. 단순히 PIR 센서의 움직임만으로 침입을 판단하면 사람 외의 움직임에도 반응할 수 있다. 그래서 PIR 센서를 1차 트리거로 사용하고, USB 웹캠과 OpenCV DNN MobileNet SSD 모델을 2차 확인 단계로 사용한다.

PIR 센서가 움직임을 감지하면 웹캠 프레임을 가져오고, AI 모델이 프레임 안의 person 객체를 찾는다. confidence가 기준값 이상이면 침입 이벤트로 판단한다. 이후 능동 부저를 울리고, 감지 이미지를 저장하고, 텔레그램 메시지 및 사진을 전송하며, CSV 로그 파일에 이벤트를 기록한다.

이 프로젝트의 AI 기능은 얼굴이 누구인지 식별하는 얼굴인식이 아니라, 수업에서 사용한 OpenCV DNN MobileNet SSD 기반 사람 객체검출이다.

2. 해결하려는 문제

  • 방에 누군가 들어왔는지 자동으로 감지한다.
  • 단순 움직임 감지의 오탐을 줄이기 위해 AI 사람 객체검출을 추가한다.
  • 침입 이벤트를 로컬 경고음, 원격 텔레그램 알림, CSV 로그로 남긴다.
  • cooldown 기능으로 같은 상황에서 알림이 너무 많이 반복되지 않게 한다.
  • MOCK_MODE로 하드웨어 없이도 코드 흐름을 테스트할 수 있게 한다.

5. 처리 순서 구조도

  1. main.py가 실행되고 시스템 준비 메시지를 출력한다.
  2. 무한 루프에서 sensor.read_motion()으로 PIR 센서 값을 읽는다.
  3. PIR 값이 움직임 감지이면 vision_ai.detect_person()을 호출한다.
  4. vision_ai.py가 USB 웹캠 프레임을 읽고 OpenCV DNN 모델을 실행한다.
  5. 모델 결과 중 class id가 1인 person 객체를 찾는다.
  6. confidence가 PERSON_CONFIDENCE_THRESHOLD 이상이면 침입 이벤트로 판단한다.
  7. cooldown 시간이 지나지 않았다면 중복 알림을 막고 루프로 돌아간다.
  8. cooldown이 끝났다면 이미지를 저장하고 부저를 울린다.
  9. 텔레그램 메시지와 사진을 전송한다.
  10. logger.write_event()가 CSV 로그를 기록한다.
  11. Ctrl+C 또는 OpenCV 창의 q 입력 시 카메라와 부저를 정리하고 종료한다.

6. 모듈 구조도

main.py 전체 프로그램 진입점이다. PIR 감지, AI 확인, cooldown, 부저, 텔레그램, 로그 저장 흐름을 연결한다.
config.py GPIO 핀, 모델 경로, confidence 기준, 텔레그램 정보, 저장 경로, MOCK_MODE, 카메라 표시 옵션을 관리한다.
sensor.py PIR 센서 값을 읽는다. 실제 모드에서는 gpiozero의 MotionSensor를 사용하고, mock 모드에서는 테스트 값을 반환한다.
buzzer.py 능동 부저를 켜고 끈다. 침입 이벤트 발생 시 beep(duration)으로 경고음을 출력한다.
vision_ai.py USB 웹캠 프레임을 읽고 OpenCV DNN MobileNet SSD 모델로 person 객체를 탐지한다. 카메라 미리보기 창과 이미지 저장도 담당한다.
telegram_notifier.py Telegram Bot API로 텍스트 메시지와 사진을 보낸다. token 또는 chat_id가 없으면 전송을 건너뛴다.
logger.py logs/event_log.csv에 이벤트를 기록한다. 파일이 없으면 헤더를 먼저 만든다.
download_models.py OpenCV DNN 모델 파일을 직접 다운로드하는 대체 스크립트이다. 수업자료 clone 방식이 안 될 때 사용한다.
requirements.txt 실행에 필요한 Python 라이브러리 목록이다. OpenCV, numpy, gpiozero, requests가 포함된다.

9. 설정 데이터 구조

설정값 기본값 역할
PIR_SENSOR_PIN 16 PIR Signal GPIO 번호
BUZZER_PIN 18 능동 부저 제어 GPIO 번호
PERSON_CONFIDENCE_THRESHOLD 0.50 사람 객체검출 기준값
ALERT_COOLDOWN_SECONDS 10.0 중복 알림 방지 시간

MOCK_MODE True 하드웨어 없이 테스트할지 여부

11. 이벤트 판단 로직

if PIR 센서가 움직임을 감지하지 않음:
    계속 대기

if PIR 센서가 움직임을 감지함:
    USB 웹캠 프레임 읽기
    OpenCV DNN으로 person 객체검출

if person confidence < 기준값:
    침입으로 판단하지 않음

if person confidence >= 기준값:
    cooldown 확인
    cooldown 중이면 중복 알림 생략
    cooldown이 아니면 침입 이벤트 처리
        - 이미지 저장
        - 부저 경고음
        - 텔레그램 메시지/사진 전송
        - CSV 로그 저장

12. 보고서용 기획 설명 문장

본 프로젝트는 PIR 인체감지 센서와 OpenCV DNN MobileNet SSD 객체검출을 결합한 Raspberry Pi 기반 AIoT 방 침입 감지 시스템이다. PIR 센서는 움직임을 빠르게 감지할 수 있지만, 사람 외의 움직임에도 반응할 가능성이 있다. 따라서 본 시스템은 PIR 센서를 1차 감지 장치로 사용하고, USB 웹캠과 AI 사람 객체검출을 2차 확인 단계로 사용하여 침입 판단의 신뢰도를 높였다.

침입으로 판단되면 능동 부저를 울려 현장에서 경고음을 출력하고, 텔레그램 봇 API를 통해 사용자에게 메시지와 사진을 전송한다. 또한 모든 이벤트는 CSV 파일에 시간, 이벤트 종류, 메시지, AI confidence, 이미지 경로와 함께 저장된다. 이 구조는 센서 입력, AI 영상 분석, GPIO 출력, 네트워크 알림, 데이터 기록을 함께 활용하므로 AIoT 수업에서 학습한 여러 기술을 통합적으로 적용한 사례이다.

13. 장점과 한계

구분 내용
장점 PIR 센서만 쓰는 방식보다 AI 사람 객체검출을 추가하여 침입 판단 신뢰도를 높일 수 있다.
장점 텔레그램 메시지와 사진을 통해 사용자가 원격으로 상황을 확인할 수 있다.
장점 CSV 로그와 저장 이미지를 통해 결과보고서와 시연 자료를 만들기 쉽다.
한계 조명, 카메라 각도, 모델 파일 경로, confidence 기준값에 따라 탐지 성능이 달라질 수 있다.
한계 OpenCV 창 표시는 라즈베리파이 Desktop 환경이 필요하며 SSH 환경에서는 제한될 수 있다.
